# 政策执行评估框架

## 概述
在推行重大政策前，必须系统评估实施条件、潜在风险和社会影响，综合考虑政策目标、执行条件、民生影响、社会稳定及政治风险，避免机械执行或盲目推行导致严重后果。

## 评估维度

### 一、政策目标分析
- 政策的核心要求是什么？
- 必须完成的硬性指标有哪些？
- 完成期限和数量要求？

### 二、实施条件评估
- 时间窗口是否合理？
- 资源配置是否到位？
- 技术条件是否成熟？
- 配套措施是否完善？

### 三、民生影响评估
- 政策执行对百姓生计的影响？
- 是否会造成粮食短缺或大规模失业？
- 百姓是否有替代生存方案？
- 基本生存需求能否保障？

### 四、社会稳定风险评估
- 可能引发多少不满民众？是否会引发群体性事件或民变？
- 地域特殊性（如动荡地区）是否考虑？
- 风险与收益是否成比例？

### 五、政治风险评估
- 是否会被政治势力利用？
- 党争因素对政策的影响？
- 执行者的立场和处境风险？

## 决策流程

1. **政策必须办** → 确认执行
2. **条件是否具备？**
   - 具备 → 正常执行
   - 不具备 → 先解决前置条件或分阶段实施
3. **民生是否有保障？**
   - 有保障 → 推进执行
   - 无保障 → 先解决粮食/生计问题，完善配套保障
4. **风险是否可控？**
   - 可控 → 执行
   - 不可控 → 上报请求支援或调整

**决策逻辑参考：**
- 若民生影响大 + 社会风险高 → 建议暂缓或调整
- 若实施条件不足 → 建议分阶段实施
- 若政治风险高 → 建议明确责任边界

## 缓解策略与配套措施
- **分阶段实施**：合理规划时间节点（如将三年任务分摊）。
- **完善保障**：建立风险预警机制，提供粮食借贷等配套保障。
- **向上管理**：向上级如实汇报困难，请求资源支持。
- **责任界定**：明确责任边界，避免成为政治斗争牺牲品。

## 典型案例警示
**案例背景**：推行改稻为桑政策。
**问题**：桑苗插下去到秋后也没有几片嫩叶养中秋晚秋的蚕；官府不借贷粮食，只叫稻农把稻田改了，秋后便没有饭吃。
**后果**：如果为了多产三十万匹丝绸，出了三十万个反民，一颗人头交代不下来。
**教训**：先解决百姓生计问题，再推行政策；对囤积居奇者施压，而非逼迫百姓。

## 预期结果
形成全面的风险评估报告，在完成政策目标的同时，保障民生稳定，避免引发社会动荡或政治危机。
